# AI Dev Agents

[🇪🇸 Leer en español](README.es.md)

Specialized agent system for AI-assisted development. Language, framework and AI tool agnostic. Supports Clean Architecture, MVC, MVVM and None (~90% of all software projects).

## What Is This

12 specialized agents + 1 orchestrator that guide AI assistants during software development. Includes a cross-verification pattern between agents to prevent errors that a single agent cannot detect.

**Key principle:** Agents define WHAT to verify (agnostic Quality Gates). Skills define HOW to do it (framework-specific).

<details>
<summary><strong>New here? Start here</strong></summary>

### Step 1: Try one agent

Pick a task you're already doing (e.g., writing a test) and invoke `@tdd-developer`. See if the gates catch something you'd miss. No setup beyond Quick Start needed.

### Step 2: Add your project context

Fill `.ai/agents/project-context.md` with your domain, constraints and stack decisions. Run the bootstrap prompt (`.ai/prompts/bootstrap.md`) to generate skills for your stack. Agents go from generic to project-aware.

### Step 3: Use the orchestrator

For features that touch multiple layers, invoke `@orchestrator` instead of individual agents. It routes to the right agents and runs cross-verification. This is where the system pays off — but it requires Step 2 to be effective. Only available in tools with on-demand agents (Claude Code, OpenCode, Antigravity, Gemini CLI, Codex CLI). For other tools (Cursor, Windsurf, Copilot), use [Feature Specs](.ai/templates/FEAT-TEMPLATE.md) to centralize context — agents are loaded inline and extract what they need from the spec.

> Full walkthrough: [getting-started guide](.ai/docs/getting-started.md) (5 minutes).

</details>

## Tool Compatibility

| Tool | Always-loaded file | On-demand agents | Token savings |
|---|---|---|---|
| Claude Code | `CLAUDE.md` | `.claude/agents/` (yes) | High |
| OpenCode | `AGENTS.md` | `.opencode/agents/` (yes) | High |
| Antigravity (Google) | `GEMINI.md` | `.agents/rules/` (yes, @mention) | High |
| Gemini CLI | `GEMINI.md` | `.gemini/skills/` (yes, activate_skill) | High |
| Codex CLI (OpenAI) | `AGENTS.md` | `.agents/skills/` (yes, /skills) | High |
| Continue (VS Code) | `.continue/rules/` | Partial (implicit matching) | Medium |
| Cursor | `.cursorrules` | No (all inline) | Low |
| Windsurf | `.windsurfrules` | No (all inline) | Low |
| GitHub Copilot / Copilot CLI | `.github/copilot-instructions.md`, `AGENTS.md` | CLI: yes (AGENTS.md) / IDE: no | CLI: High / IDE: Low |

Tools with on-demand agents/skills (Claude Code, OpenCode, Antigravity, Gemini CLI, Codex CLI) load agents ONLY when invoked — high token savings. Tools without on-demand agents load everything in a compact file. For those, **[SDD (Feature Specs)](.ai/templates/FEAT-TEMPLATE.md) is strongly recommended** to centralize context.

> See [supported tools & downloads](.ai/docs/supported-tools.md), [detailed tool capabilities](.ai/docs/how-claude-code-works.md) and [cross-verification patterns](.ai/docs/cross-verification.md) for more.

## Quick Start

### 1. Add `.ai/` to your project

**Option A:** Click "Use this template" on the GitHub repo (full repo clone).

**Option B:** Download only `.ai/` into an existing project (requires Node.js)

```bash
npx degit luismiglezmohino/ai-dev-agents/.ai .ai
```

**Option C:** Without Node.js

```bash
curl -sL https://github.com/luismiglezmohino/ai-dev-agents/archive/main.tar.gz \
  | tar xz && mv ai-dev-agents-main/.ai . && rm -rf ai-dev-agents-main
```

> Just [`.ai/`](.ai/) — everything else is generated by `sync.sh` in step 3.

### 2. Configure project

**Option A: Automatic bootstrap (recommended)**

Open your AI tool and paste the content of `.ai/prompts/bootstrap.md`.
The LLM analyzes your project (or asks if new) and automatically generates:
- `.ai/agents/project-context.md` — filled project context
- `CLAUDE.md` — main configuration
- `.ai/decisions.md` — initial stack decisions
- `.ai/skills/` — basic stack skills

**Option B: Manual** — copy templates and fill placeholders.

### 3. Sync and validate

```bash
.ai/sync.sh   # Generates configs for all 9 tools
.ai/test.sh   # Validates structure (0 errors = ready)
```

> Always run `sync.sh` after any change in `.ai/`. It generates `AGENTS.md`, `.claudeignore`, `docs/` structure, `.gitignore` entries, and tool-specific configs.

### What to commit vs gitignore

| File/Directory | Commit? | Why |
|---|---|---|
| `.ai/` (entire directory) | **Yes** | Source of truth |
| `CLAUDE.md` | **Yes** | Main config |
| `AGENTS.md` | **Yes** | Generated by sync.sh, but commit it |
| `.claude/settings.json` | **Yes** | Hooks + MCP config |
| `docs/` | **Yes** | Specs, ADRs (created by sync.sh) |
| `.claude/agents/`, `.claude/rules/`, `.opencode/` | **No** | Generated by `sync.sh` |
| `.cursorrules`, `.windsurfrules`, `GEMINI.md` | **No** | Generated by `sync.sh` |
| `.agents/` | **No** | Generated by `sync.sh` (Codex, Gemini CLI, Antigravity) |
| `.continue/rules/` | **No** | Generated by `sync.sh` (Continue) |
| `.gemini/skills` | **No** | Generated by `sync.sh` (Gemini CLI) |
| `.ai/.local/` | **No** | Session state |

> **Rule:** if `sync.sh` generates it, gitignore it. If you edit it by hand, commit it. Always edit in `.ai/` and re-run `sync.sh` to propagate changes. Sync generates for all 9 tools at once — this allows switching tools without reconfiguring.

## Architecture Support

The template supports **4 architecture modes** (~90% of all software projects):

| Flag | When to use | Agents adapted |
|---|---|---|
| `Clean` (default) | Backend with separated layers (Domain/Application/Infrastructure) | None (default gates) |
| `MVC` | Framework MVC (Laravel, Django, Rails, Spring MVC...) | architect, tdd-developer, qa-engineer, orchestrator |
| `MVVM` | Mobile apps (Android, iOS, Flutter) | architect, ux-designer, performance-engineer, orchestrator |
| `None` | Frontend-only SPA, CLI tools, or no formal architecture | Architect not invoked |

The bootstrap detects the architecture from folder structure or asks. It adapts the relevant agents — the rest stay unchanged.

> Web frontend (Vue/React) uses `None` — covered by ux-designer + skills, not by architect gates.

## Reusable Prompts

| Prompt | When to use | Required |
|---|---|---|
| `bootstrap.md` | Day 0 — generates configs + skills | Yes |
| `feature-spec.md` | Before complex features (SDD) | No |
| `code-review.md` | Before opening a PR | No |
| `refine-skills.md` | After 2-3 features | No |
| `legacy-audit.md` | Before modernizing legacy code | No |

All prompts available in English (`.ai/prompts/`) and Spanish (`.ai/prompts/es/`). Architecture-specific gates (`gates-mvc.md`, `gates-mvvm.md`) are loaded on demand by the bootstrap — no need to paste them separately.
